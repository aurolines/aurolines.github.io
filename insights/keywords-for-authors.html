Why keywords matter (quick)

Keywords + categories are the primary signal Amazon uses to surface books. Good keywords = more impressions; long-tail = higher relevance and click-through in India (regional searches, genre + utility queries).

Use these templates to create long-tail phrases: replace {GENRE}, {HOOK}, {AUDIENCE}, {LOCATION}.

"{GENRE} {HOOK} novel" — e.g. "contemporary romance college love novel"
"{GENRE} {AUDIENCE} ebook" — e.g. "romantic poetry young adults ebook"
"best {GENRE} for {LOCATION}" — e.g. "best poetry for india"
"{HOOK} {GENRE} book" — e.g. "office romance contemporary novel"

Publishing Platforms tips — where to paste keywords

  • Primary: Title + Subtitle (use 1–2 strong long-tail phrases naturally).
  • Backend keywords: Use 7 slots — include variations and commas are not needed; do not repeat words from title/subtitle.
  • Categories: Pick the most specific 2 categories; manually request new categories via Publishing Platforms support if necessary.

India-specific keyword notes

  • Include regional signals: "India", city names (for travel/local non-fiction), or cultural hooks (e.g., "arranged marriage") where relevant.
  • Use INR price queries in descriptions occasionally for local buyers (helps CTR).
  • Hindi/English mix searches exist — test a Hindi phrase if your audience uses it.

FAQ (SEO-friendly)

How many keywords should I use on Publishing Platforms?
Publishing Platforms allows seven backend keyword fields — use all of them. Prefer long-tail phrases over single words.
Should I include genre names or phrases?
Yes — include the genre, then add modifiers (hook, audience, location). Focus on relevancy first.

Tools & next steps

Use Search Console to find queries that already get impressions — optimize titles/descriptions for those. Export Publishing Platforms search terms and iterate every 4–6 weeks.

Free: Google Keyword Planner (seed ideas), Search Console (queries)
Low-cost: Publishing Platforms Rocket alternatives, MerchantWords (for validation)
